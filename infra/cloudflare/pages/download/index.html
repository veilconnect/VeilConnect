自己部署(推荐给重视隐私的用户)

在一台有公网 IP 的 Linux 服务器上,一条命令即可(自动装 Docker、克隆代码、签发 HTTPS 证书、起服务):

curl -fsSL https://raw.githubusercontent.com/veilconnect/VeilConnect/main/scripts/bootstrap.sh | sudo bash -s -- chat.example.com

局域网/无域名先本机试用:把末尾换成 --local。完整说明见 自部署指南。

安全与信任(请务必读懂)

  • 端到端加密:消息走 Double Ratchet(Signal 协议),每条消息一把密钥、用后即删(前向保密);服务器只搬运,读不到内容。
  • 防中间人:连上后双方会得到一个安全码,需当面/电话核对一致(或用配对码自动校验)后才放行消息。请务必核对。
  • 网页版的信任边界(诚实告知):你打开本站,就需要信任 veilconnect.org 下发的网页代码。我们用 SRI(子资源完整性)让"脚本被篡改"可被浏览器检测,但无法完全消除。若你不想信任本站,请自己部署或使用签名桌面版。
  • 元数据:内容加密,但"谁在何时连接、IP"等元数据仍可能被中继方看到。高威胁场景请叠加 VPN/Tor。
